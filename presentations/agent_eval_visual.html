01

Evaluation is a quality control system.

A normal app test asks, "did the function return the expected value?" An agent evaluation asks a harder question: "did a moving multi-step system produce a useful, grounded, safe, and efficient result?"

INPUT

User question

The original task, including hidden complexity such as multiple requirements or sensitive topics.

RUN

Agent pipeline

Planner, Searcher, Reader, Writer, Critic, and optional Compliance collaborate.

STATE

Evidence trail

The run produces sources, facts, iterations, LLM calls, duration, and a final answer.

JUDGE

Evaluation suite

Specialized metrics grade different failure modes instead of using one vague score.

GATE

Pass or fail

Production systems should block release when critical quality gates fail.

Why exact match breaks

AI can answer correctly using different words. "Python is readable and versatile" and "Python is a flexible language known for clarity" are not identical strings, but they may be semantically equivalent.

Exact match is brittleMeaning matters

Why one score is dangerous

A response can be fluent but unsupported, factually correct but non-compliant, compliant but incomplete, or complete but wastefully expensive. Production evaluation separates these dimensions.

Each metric catches a different failure.

Your project uses DeepEval-style metrics and G-Eval rubrics to evaluate lexical alignment, code correctness, hallucination, and faithfulness. The important teaching point is not the library call; it is the production habit of mapping every risk to a measurable gate.

DeepEval and G-Eval in plain English

DeepEval is the evaluation harness: it organizes test cases, metrics, scores, pass/fail decisions, and reasons. G-Eval is the rubric-driven judge pattern: it asks a model to grade an open-ended answer against explicit criteria.

03

Compliance turns safety into release criteria.

In the evaluation folder, compliance checks are framed around EU AI Act style prohibited-risk behavior and Indian DPDP style personal-data leakage. Students should understand these as product risk lenses, not just test names.

EU AI Act lens

The grader looks for outputs that drift toward prohibited or high-risk behavior: manipulation, exploitation of vulnerable people, social scoring, unauthorized profiling, or workplace emotion recognition.

  • Teaching idea: AI safety includes what the system refuses to do.
  • Production habit: Fail risky behavior before launch, not after a user reports harm.

Indian DPDP lens

The grader checks whether personal identifiers are exposed in plain text. Aadhaar, PAN, voter IDs, passport numbers, phone numbers, and emails should be masked or redacted when they do not belong in the answer.

  • Teaching idea: Privacy is not only storage; generated text can leak data too.
  • Production habit: Use placeholders that preserve meaning while removing sensitive values.

Compliance is defense in depth

One prompt instruction is not enough. A robust system uses input policy, agent routing, output checks, evaluation tests, logs, and human escalation for sensitive cases. The evaluation layer is the audit gate that tells you whether the rest of the system behaved.

04

Agent evaluation must inspect the journey.

A final answer can hide operational problems. Your live test runs the real LangGraph pipeline, extracts the final state, and then evaluates the answer with the real context the agents gathered.

1

Compile team

The selected agents become the active graph for this question.

2

Invoke graph

The system runs real planning, search, writing, critique, and possible revision.

3

Capture state

Sources, facts, iterations, call count, and elapsed time become evaluation evidence.

4

Build context

Source snippets and extracted facts become the retrieval context for factual judging.

5

Report gates

Every metric returns a score, pass/fail status, and reason.

Step efficiency

Did the agent solve the task in a reasonable number of steps, or did the writer and critic loop wastefully?

Task completion

Did the answer satisfy every part of the prompt, especially multi-part requests?

Cost awareness

LLM calls, search calls, iterations, and duration are product signals, not trivia.

05

Vocabulary and final recap.

These are the words students should be able to use correctly after the lesson.

Evaluation

Measuring whether an AI system meets a defined quality standard.

Metric

A specific scoring rule for one behavior, such as faithfulness or compliance.

DeepEval

An evaluation framework for organizing AI test cases, metrics, thresholds, and reports.

Rubric

Plain-language grading criteria used by a judge model.

G-Eval

A rubric-based evaluation technique for grading open-ended model outputs.

LLM-as-Judge

A stronger model or configured model used to grade open-ended AI outputs.

Hallucination

A generated claim not supported by the available evidence.

Faithfulness

How closely the answer stays loyal to retrieved facts and source context.

PII

Personally identifiable information such as national IDs, phone numbers, and emails.

Threshold

The minimum acceptable score for a release gate.

Trace

The full record of agent steps, tool use, loops, timing, and final state.

Final summary

  • AI evaluation is not one test; it is a layered quality system.
  • Exact match is useful for deterministic values, but semantic tasks need rubric-based judgment.
  • Factual quality has two sides: avoid invented claims and stay faithful to retrieved evidence.
  • Compliance checks convert legal and privacy expectations into automated release gates.
  • Agent systems must be evaluated from their trace, because cost, loops, tool behavior, and completion matter in production.
